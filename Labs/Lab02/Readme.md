### Лабараторная работа № 3. Расчет подсетей IPv4

## Задачи

- Определение подсетей по IPv4-адресу
- Расчет подсетей по IPv4-адресу

|Дано| |
|----|-|
|IP-адрес узла|192.168.200.139|
|Исходная маска подсети|255.255.255.0|
|Новая маска подсети|255.255.255.224|

|Найти| |
|-----|-|
|Количество бит подсети| |
|Количество созданных подсетей| |
|Количество бит узлов в подсети| |
|Количество узлов в подсети| |
|Сетевой адрес этой подсети| |
|IPv4-адрес первого узла в этой подсети| |
|IPv4-адрес последнего узла в этой подсети| |
|Широковещательный IPv4-адрес в этой подсети| |

решение:

Сначала переведём в двоичную систему счисления ip адрес узла. Для этого воспользуемся таблицей:

|128|64|32|16|8|4|2|1|
|---|--|--|--|-|-|-|-|
|$$2^7$$|$$2^6$$|$$2^5$$|$$2^4$$|$$2^3$$|$$2^2$$|$$2^1$$|$$2^0$$|
|1|2|3|4|5|6|7|8|

Каждый столбец соответствует биту в ip адресе. Десятичное число складывается из сложения единиц в октете. Положение каждого бита соответствует совему числу. 
Так например число `255` складывется из восьми единиц в октете `11111111`, число `80` из `01010000`.

Сначала переведём в двоичную систему счисления ip адрес узла и маски.

|ip|11000000.10101000.11001000.10001011|
|--|-----------------------------------|
|mask|11111111.11111111.11111111.00000000|
