### Лабораторная работа № 3. Расчет подсетей IPv4

## Задачи

- Определение подсетей по IPv4-адресу
- Расчет подсетей по IPv4-адресу

|Дано| |
|----|-|
|IP-адрес узла|192.168.200.139|
|Исходная маска подсети|255.255.255.0|
|Новая маска подсети|255.255.255.224|

|Найти| |
|-----|-|
|Количество бит подсети| |
|Количество созданных подсетей| |
|Количество бит узлов в подсети| |
|Количество узлов в подсети| |
|Сетевой адрес этой подсети| |
|IPv4-адрес первого узла в этой подсети| |
|IPv4-адрес последнего узла в этой подсети| |
|Широковещательный IPv4-адрес в этой подсети| |

## Решение:

Сначала переведём в двоичную систему счисления IP-адрес узла. Для этого воспользуемся таблицей:

|128|64|32|16|8|4|2|1|
|---|--|--|--|-|-|-|-|
|$$2^7$$|$$2^6$$|$$2^5$$|$$2^4$$|$$2^3$$|$$2^2$$|$$2^1$$|$$2^0$$|
|1|2|3|4|5|6|7|8|

Каждый столбец соответствует биту в IP-адресе. Десятичное число складывается из сложения единиц в октете. Положение каждого бита соответствует своему числу. 
Так, например, число `255` складывается из восьми единиц в октете `11111111`, число `80` - из `01010000`.

Сначала переведём в двоичную систему счисления IP-адрес узла и маски.

| | |
|-|-|
|IP|11000000.10101000.11001000.10001011|
|Маска|11111111.11111111.11111111.00000000|

Далее мы можем применить логическую операцию "И". В результате узнаем сеть узла.

Таблица значений при применении операции "И":

```
0 И 0 = 0
1 И 0 = 0
0 И 1 = 0
1 И 1 = 1
```

В итоге мы выясняем, что узел 192.168.200.139 принадлежит сети 192.168.200.0/24.

/24 означает, сколько старших битов выделено под маску.

Используя таблицу, вычислим новую маску:

```
255 = 128+64+32+16+8+4+2+1

224 = 128+64+32

получаем

11111111.11111111.11111111.11100000
```


3 старших бита четвёртого октета используются в новой маске. Соответственно, для подсети будет использоваться 27 бит в маске.

Зная количество бит в новой маске, можно определить, сколько подсетей можно создать.

Рассчитывается по формуле $$2^k$$, где \(k\) - количество бит, добавленных в новую маску.

Получаем:

```
2^3=8
```

8 подсетей.

Рассчитаем количество узлов в подсети по формуле $$2^n - 2$$, где \(n\) — это количество бит, выделенных под идентификатор хоста.

Как мы знаем, в октете 8 бит. Для подсети выделено 3 бита, 5 бит остаётся для хостов.

Получаем:

```
2^5-2=30
```

30 узлов в подсети.

Вычислим сетевой адрес подсети:

| | |
|-|-|
|IP узла|11000000.10101000.11001000.10001011|
|Маска подсети|11111111.11111111.11111111.11100000|

Получаем сетевой адрес: **192.168.200.128/27**.

Вычислим широковещательный адрес для подсети, заполнив единицами все биты хостовой части. Получим адрес: **192.168.200.159**.

Зная сетевой адрес и широковещательный адрес, мы можем определить первый и последний адрес в подсети. Это будут **192.168.200.129** и **192.168.200.158**.

|Итог| |
|----|-|
|Количество бит подсети|27|
|Количество созданных подсетей|8|
|Количество бит узлов в подсети|5|
|Количество узлов в подсети|30|
|Сетевой адрес этой подсети|192.168.200.128/27|
|IPv4-адрес первого узла в этой подсети|192.168.200.129|
|IPv4-адрес последнего узла в этой подсети|192.168.200.158|
|Широковещательный IPv4-адрес в этой подсети|192.168.200.159|
